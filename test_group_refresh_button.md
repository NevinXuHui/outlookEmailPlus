# 分组批量刷新Token功能测试

## 功能概述

在账号列表工具栏添加"🔄 刷新Token"按钮，可以一键刷新当前分组内所有账号的OAuth Token。

## UI位置

```
┌──────────────────────────────────────────────────────────┐
│ 排序: [🕐刷新时间] [📧邮箱名] │ ☑️⚠️异常 │ [🔄刷新Token] │ □全选 │
├──────────────────────────────────────────────────────────┤
│ 账号列表...                                               │
└──────────────────────────────────────────────────────────┘
```

按钮位置：在"异常"筛选和"全选"之间

## 功能流程

### 1. 点击"🔄 刷新Token"按钮

系统会执行以下步骤：

1. **验证分组**
   - 检查是否选中了分组
   - 检查是否是临时邮箱分组（临时邮箱不支持Token刷新）

2. **获取账号列表**
   - 调用 `/api/accounts/all-ids?group_id={currentGroupId}` 获取分组内所有账号ID
   - 包括当前应用的筛选条件（搜索、标签等）

3. **确认对话框**
   ```
   确认刷新「分组名」分组内的 X 个账号的 Token？
   
   注意：IMAP 账号将自动跳过，此操作可能需要较长时间。
   ```

4. **执行刷新**
   - 调用现有的 `batchRefreshSelected()` 函数
   - 使用SSE流式显示进度
   - 自动跳过IMAP账号

5. **实时反馈**
   - 显示持久化Toast进度条
   - 实时更新刷新进度：`🔄 正在刷新 Token... X / Y`
   - 完成后自动刷新账号列表

## 与现有功能的区别

| 功能 | 位置 | 操作对象 | 适用场景 |
|------|------|----------|----------|
| **新增：分组刷新** | 工具栏 | 当前分组所有账号 | 快速刷新整个分组 |
| 批量操作栏刷新 | 底部批量操作栏 | 选中的账号 | 精确控制要刷新的账号 |

## 代码实现

### 前端修改

**文件**: `templates/index.html`
- 在工具栏添加按钮：
  ```html
  <button class="btn btn-sm btn-primary" 
          onclick="refreshCurrentGroupTokens()" 
          title="批量刷新当前分组所有账号的Token">
      🔄 刷新Token
  </button>
  ```

**文件**: `static/js/main.js`
- 新增函数 `refreshCurrentGroupTokens()`：
  - 验证当前分组
  - 获取分组所有账号ID
  - 调用现有的 `batchRefreshSelected()` 执行刷新

### 后端API

复用现有API端点：
- `GET /api/accounts/all-ids?group_id={id}` - 获取分组所有账号ID
- `POST /api/accounts/refresh/selected` - 批量刷新Token（SSE流式）

## 测试用例

### 测试1: 未选择分组
- **操作**: 未选择分组时点击"🔄 刷新Token"
- **预期**: 显示提示"请先选择一个分组"

### 测试2: 临时邮箱分组
- **操作**: 选择"临时邮箱"分组后点击按钮
- **预期**: 显示提示"临时邮箱分组不支持 Token 刷新"

### 测试3: 正常分组
- **操作**: 选择普通分组（如"默认分组"）后点击按钮
- **预期**: 
  1. 显示确认对话框，包含分组名和账号数量
  2. 确认后开始刷新
  3. 显示进度Toast
  4. 完成后自动刷新列表

### 测试4: 空分组
- **操作**: 选择没有账号的分组后点击按钮
- **预期**: 显示提示"当前分组没有可刷新的账号"

### 测试5: 包含IMAP账号的分组
- **操作**: 刷新包含IMAP账号的分组
- **预期**: 
  - IMAP账号被自动跳过
  - 只刷新Outlook账号
  - Toast显示实际刷新的账号数

## 边界情况处理

1. **网络错误**: 显示"获取分组账号失败，请重试"
2. **用户取消**: 点击确认对话框的"取消"后不执行任何操作
3. **刷新超时**: 由现有的超时机制处理（2分钟整体超时，30秒心跳超时）
4. **部分失败**: SSE流式显示每个账号的刷新结果

## 用户体验优化

1. **快速访问**: 无需选中账号即可操作，提升效率
2. **明确提示**: 确认对话框显示分组名和账号数，避免误操作
3. **实时反馈**: SSE流式显示进度，用户可随时了解刷新状态
4. **自动更新**: 刷新完成后自动重新加载账号列表
5. **智能跳过**: 自动识别并跳过不支持Token刷新的IMAP账号

## 技术亮点

- ✅ 复用现有的批量刷新基础设施（SSE、超时控制、错误处理）
- ✅ 无需修改后端代码，完全通过前端组合实现
- ✅ 与筛选条件无缝集成（搜索、标签、异常筛选等）
- ✅ 保持与现有批量刷新功能的一致性
